## Анализ

### QNetwork (базовый DQN)
- **Лучший результат** — стабильно достигает максимума 500
- Самое быстрое обучение
- Минимальное число параметров (17,410)
- Идеально для простых задач как CartPole

### DuelingQNetwork
- Улучшился с 126.4 (50 эпох) до 176.8 (100 эпох)
- Требует больше времени для сходимости
- Для CartPole архитектура избыточна

### QNetworkBN
- **Неожиданно плохой результат** (20.1 при 100 эпохах vs 403.7 при 50)
- BatchNorm не стабилен для RL задач
- Вероятно переобучение или неправильная инициализация

### QNetworkLSTM
- Улучшился с 224.8 до 235.5
- Для CartPole временные зависимости не важны
- 68k параметров — избыточно для простой задачи

## Выводы

1. **QNetwork** — лучший выбор для CartPole-v1
2. Сложные архитектуры (LSTM, Dueling, BN) не дают преимуществ для простых задач
3. BatchNorm требует дополнительной настройки для RL
