# Сравнение архитектур DQN

## Параметры обучения

| Параметр | Значение |
|----------|----------|
| Среда | CartPole-v1 |
| Эпизоды | 50 |
| Размер скрытого слоя | 128 |
| Размер батча | 32 |
| Learning rate | 0.003 |
| Gamma | 0.99 |
| Буфер | 10000 |
| Устройство | CUDA (GTX 1650 Ti) |

## Результаты

| Архитектура | Время (сек) | Средняя награда (оценка) | Максимум | Минимум | Параметры |
|-------------|-------------|---------------------------|----------|---------|-----------|
| **QNetwork** (базовый DQN) | 32.9 | **500.0** | 500.0 | 500.0 | 17,410 |
| QNetworkBN (с BatchNorm) | 27.4 | 403.7 | 500.0 | 167.0 | 17,922 |
| QNetworkLSTM | 29.1 | 224.8 | 266.0 | 195.0 | 68,866 |
| DuelingQNetwork | 19.7 | 126.4 | 133.0 | 118.0 | 33,859 |

## Выводы

1. **QNetwork (базовый DQN)** — лучший результат, достиг максимума 500 (предел CartPole-v1). Простая и эффективная архитектура.

2. **QNetworkBN** (BatchNorm + Dropout) — второй результат (403.7), но нестабилен. Требует больше эпизодов для стабилизации.

3. **QNetworkLSTM** — хуже ожиданий (224.8). Для CartPole временные зависимости не важны, LSTM избыточен. 68k параметров — много для простой задачи.

4. **DuelingQNetwork** — худший результат (126.4). Для простой задачи CartPole разделение V/A избыточно, требует тонкой настройки.

## Рекомендации

Для CartPole-v1 рекомендуется **базовый QNetwork** — он проще, стабильнее и достигает оптимальных результатов с наименьшим числом параметров.
