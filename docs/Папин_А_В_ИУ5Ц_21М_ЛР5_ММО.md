## 4. Теоретические сведения

### 4.1 Алгоритм DQN

Deep Q-Network (DQN) — это алгоритм обучения с подкреплением, который использует глубокую нейронную сеть для аппроксимации Q-функции.

**Основные компоненты:**

1. **Q-Network** — нейронная сеть, аппроксимирующая Q-значения Q(s, a)
2. **Target Network** — копия Q-network, используемая для стабилизации обучения
3. **Experience Replay** — буфер для хранения переходов (s, a, r, s')
4. **Epsilon-greedy** — стратегия исследования

**Алгоритм:**
1. Инициализировать Q-network и target network
2. Заполнить буфер опыта случайными действиями
3. For each episode:
   a. Получить состояние s
   b. Выбрать действие a (epsilon-greedy)
   c. Выполнить действие, получить (r, s')
   d. Сохранить переход в буфер
   e. Если достаточно данных в буфере:
      - Случайная выборка батча
      - Вычислить target: y = r + gamma * max_a' Q_target(s', a')
      - Обновить Q-network (MSE loss)
   f. Каждые target_update шагов: обновить target network

### 5.1 QNetwork (базовый DQN)

```
QNetwork(
  (fc1): Linear(4 -> 128)
  (fc2): Linear(128 -> 128)
  (fc3): Linear(128 -> 2)
)
Параметров: 17 410
```

### 5.2 DuelingQNetwork

```
DuelingQNetwork(
  (feature): Linear(4 -> 128)
  (fc_val): Linear(128 -> 128)
  (fc_adv): Linear(128 -> 128)
  (fc_value): Linear(128 -> 1)
  (fc_advantage): Linear(128 -> 2)
)
Параметров: 33 859
```

Формула: Q(s,a) = V(s) + A(s,a) - mean(A(s,a))

## 7. Сравнение архитектур

### 7.1 QNetwork (базовый DQN)
- 50 эпох: 500.0 (максимум!)
- 100 эпох: 500.0 (максимум!)
- 500 эпох: 112.3 (переобучение!)
- **Вывод:** Overfitting при долгом обучении. Лучший результат для простых задач.

### 7.2 DuelingQNetwork
- 50 эпох: 126.4
- 100 эпох: 176.8
- 500 эпох: 116.3
- **Вывод:** Нестабилен, не дает преимуществ для простых задач.

### 7.3 QNetworkBN
- 50 эпох: 403.7
- 100 эпох: 20.1
- 500 эпох: 74.6
- **Вывод:** BatchNorm нестабилен в RL задачах.

### 7.4 QNetworkLSTM
- 50 эпох: 224.8
- 100 эпох: 235.5
- 500 эпох: 154.7
- **Вывод:** Устойчивее к долгому обучению, но избыточен для CartPole.

### Ключевые наблюдения
1. 100 эпизодов — оптимально для QNetwork, дальше — переобучение
2. LSTM устойчивее к долгому обучению
3. BatchNorm показывает худшую стабильность в RL задачах
4. Dueling не дает преимуществ для простых задач

## 8. Выводы

В ходе лабораторной работы были исследованы 4 архитектуры DQN для задачи CartPole-v1:

1. **Базовая QNetwork** показала лучшие результаты (500.0 при 50-100 эпизодах) благодаря простоте и отсутствию переобучения на коротких тренировках.

2. **DuelingQNetwork** не дал преимуществ для простой задачи CartPole, где нет необходимости разделять оценку состояния и действия.

3. **QNetworkBN** продемонстрировал нестабильность — BatchNorm плохо работает с RL из-за зависимости от распределения входных данных.

4. **QNetworkLSTM** оказался устойчивее при долгом обучении (500 эпизодов), но для CartPole избыточен — задача не требует памяти о прошлых состояниях.

**Рекомендации:**
- Для простых задач (CartPole) использовать базовую QNetwork
- Оптимальное число эпизодов: 100
- Избегать BatchNorm в RL задачах без дополнительной настройки
