# Почему L-BFGS не используют в глубоком обучении?

## Введение

L-BFGS (Limited-memory Broyden-Fletcher-Goldfarb-Shanno) — мощный квазиньютоновский метод оптимизации, который показывает отличные результаты на малых и средних задачах. Однако в глубоком обучении (CNN, MLP, Transformer) практически не применяется. В этом отчёте разбираем почему.

---

## 1. Требует вычисления полного градиента

### Проблема

| Метод | Градиент | Память |
|-------|----------|--------|
| **L-BFGS** | Полный градиент по всем данным | O(mn) |
| **SGD/Adam** | Мини-батч (32-256 образцов) | O(n) |

**Формула:**
```
L-BFGS: ∇f(x) = Σᵢ ∇L(xᵢ, yᵢ)  # ВСЯ выборка (миллионы образцов!)
Adam:   ∇f(x) ≈ Σᵢ∈batch ∇L(xᵢ, yᵢ)  # Мини-батч (32-256 образцов)
```

**Пример:** Для ImageNet (1.2 млн изображений):
- L-BFGS: вычисление полного градиента = **часы** на одной итерации
- Adam: мини-батч из 256 образцов = **миллисекунды** на итерации

---

## 2. Линейный поиск (Line Search) слишком дорогой

L-BFGS требует **точного линейного поиска** на каждой итерации:

```python
# L-BFGS: нужно найти оптимальный шаг α
α* = argmin_α f(x + α·d)  # Много вычислений функции!
```

**Проблема:** Каждый шаг линейного поиска требует вычисления функции потерь на **всех данных** → очень медленно.

**Сравнение:**
```
L-BFGS итерация:
  1. Вычислить градиент (вся выборка) — 1 час
  2. Линейный поиск (5-10 вычислений функции) — 5-10 часов
  3. Обновить матрицу — быстро
  Итого: ~6-11 часов на итерацию

Adam итерация:
  1. Вычислить градиент (мини-батч) — 1 мс
  2. Обновить моменты — 0.1 мс
  3. Обновить веса — 0.1 мс
  Итого: ~1.2 мс на итерацию
```

---

## 3. Не работает со стохастичностью

L-BFGS предполагает **детерминированную** функцию:
- Градиент должен быть точным
- При шумных градиентах (мини-батч) метод **расходится**

```python
# SGD/Adam работают с шумными градиентами:
∇f_mini_batch = ∇f_true + noise  # OK для SGD/Adam

# L-BFGS: требует ∇f_true (без шума)
# При шумных градиентах:
# - Невозможно построить точную аппроксимацию Гессе
# - Линейный поиск не сходится
# - Метод расходится
```

---

## 4. Плохо масштабируется на GPU

| Характеристика | L-BFGS | SGD/Adam |
|---------------|--------|----------|
| Параллелизм | Низкий (последовательный) | Высокий |
| Использование GPU | 10-20% | 80-95% |
| Синхронизация | Частая | Редкая |

**Проблема:** L-BFGS требует частой синхронизации для точного градиента → GPU простаивает.

**Архитектура:**
```
L-BFGS на GPU:
  CPU: вычисление направления (последовательно)
  GPU: вычисление градиента (простаивает 80% времени)
  CPU: линейный поиск (последовательно)
  
Adam на GPU:
  GPU: вычисление градиента + обновление весов (95% времени)
  CPU: минимальное участие
```

---

## 5. Чувствительность к гиперпараметрам

L-BFGS требует тонкой настройки:
- Точного линейного поиска (условия Вольфе)
- Правильного выбора m (размер памяти)
- Хорошего начального приближения

**Сравнение:**
```python
# L-BFGS: много параметров
optimizer = LBFGS(
    model.parameters(),
    lr=1.0,           # нужно подбирать
    max_iter=20,      # нужно подбирать
    max_eval=25,      # нужно подбирать
    tolerance_grad=1e-7,   # критично
    tolerance_change=1e-9, # критично
    history_size=10   # нужно подбирать
)

# Adam: работает "из коробки"
optimizer = torch.optim.Adam(
    model.parameters(), 
    lr=0.001  # работает для большинства задач
)
```

---

## 6. Не подходит для online learning

В реальном времени данные поступают потоком:

| Сценарий | L-BFGS | SGD |
|----------|--------|-----|
| Новый образец | Игнорируется до следующей итерации | Сразу обновляет веса |
| Изменение распределения | Медленная адаптация | Быстрая адаптация |
| Память | Хранит историю градиентов | Не хранит историю |

---

## Сравнение на практике

| Задача | L-BFGS | Adam |
|--------|--------|------|
| **Малая задача** (n < 1000, данные в памяти) | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **Логистическая регрессия** (sklearn) | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **CNN на CIFAR-10** (50k образцов) | ⭐ (не применим) | ⭐⭐⭐⭐⭐ |
| **ResNet на ImageNet** (1.2M образцов) | ❌ (невозможно) | ⭐⭐⭐⭐⭐ |
| **Transformer (BERT, GPT)** | ❌ (невозможно) | ⭐⭐⭐⭐⭐ |

## Где L-BFGS применяют?

| Область | Применение | Почему L-BFGS |
|---------|------------|---------------|
| **Классическое ML** | Логистическая регрессия, SVM | Малые данные, точный градиент |
| **Научные вычисления** | Оптимизация в физике, химии | Детерминированные функции |
| **Компьютерное зрение** | Выравнивание изображений, bundle adjustment | Малое число параметров |
| **NLP (малые задачи)** | CRF, максимальная энтропия | Точная оптимизация важна |

**Пример из sklearn:**
```python
# sklearn использует LBFGS по умолчанию для логистической регрессии
from sklearn.linear_model import LogisticRegression
model = LogisticRegression()  # solver='lbfgs' по умолчанию
```

## Подвох L-BFGS

```
L-BFGS — это как Ferrari в пробке:
• Быстрый на гладкой дороге (малые задачи)
• Бесполезный в час пик (большие данные)
• Требует качественного топлива (точный градиент)
• Не прощает ошибок водителя (чувствителен к параметрам)

SGD/Adam — это как Toyota Corolla:
• Не самый быстрый, но надёжный
• Работает в любых условиях
• Прощает ошибки
• Масштабируется
```

## Попытки применения L-BFGS в глубоком обучении

### Исследования (2016-2024)

| Метод | Идея | Результат |
|-------|------|-----------|
| **LBFGS для fine-tuning** | Использовать L-BFGS для последних слоёв | Работает, но медленно |
| **Stochastic L-BFGS** | Аппроксимация градиента мини-батчами | Нестабилен, расходится |
| **Distributed L-BFGS** | Параллельное вычисление градиента | Сложная синхронизация |
| **LBFGS + Adam hybrid** | Adam для обучения, L-BFGS для fine-tuning | Улучшение на 1-2% |

**Вывод:** Для глубокого обучения **SGD с моментом** и **Adam** остаются золотым стандартом.

## Вывод

**L-BFGS — отличный оптимизатор для своих задач, но глубокое обучение — не его область.**

**Рекомендации:**
- Используйте **L-BFGS** для:
  - Логистической регрессии
  - Малых задач оптимизации (n < 1000)
  - Научных вычислений с точными градиентами
  
- Используйте **Adam/SGD** для:
  - Глубоких нейронных сетей
  - Больших наборов данных
  - Online learning
  - Задач с шумными градиентами
