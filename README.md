### Выполненные задачи

#### 1. Анализ пропущенных значений
- Датасет оказался **полным** — 1 200 записей, 22 признака, **0 пропусков**
- Проверено методами: `isnull().sum()`, `info()`, визуально через heatmap

#### 2. Кодирование категориальных признаков
- Применены: **Label Encoding** и **One-Hot Encoding**
- Для признака `powertrain_type` (3 уникальных значения: Electric, Petrol, Diesel):
  - Label Encoding: создана колонка с числами 0, 1, 2
  - One-Hot Encoding: созданы 3 колонки с бинарными значениями
- **Выбран One-Hot Encoding** — не вносит искусственный порядок

#### 3. Нормализация числовых признаков
- Применены: **Min-Max Scaling** и **Standardization (Z-score)**
- Для признака `ev_sales`:
  - Min-Max: диапазон [0, 1]
  - StandardScaler: среднее = 0, std = 1
- **Выбран StandardScaler** — более устойчив к выбросам
