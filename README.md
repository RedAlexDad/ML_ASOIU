### Выполненные задачи

#### 1. Масштабирование признаков (5 способов)

| Метод | Описание | Результат |
|-------|----------|-----------|
| **StandardScaler** | Z-score нормализация | среднее ≈ 0, std ≈ 1 |
| **MinMaxScaler** | Приведение к [0,1] | диапазон [0, 1] |
| **RobustScaler** | Устойчив к выбросам | использует медиану и IQR |
| **MaxAbsScaler** | Масштабирование по макс. | диапазон [-1, 1] |
| **QuantileTransformer** | Равномерное распределение | диапазон [0, 1] |

#### 2. Обработка выбросов

**Методы:**
1. **Удаление (IQR)** — удалено строк: **50** (из 1200 → 1150)
2. **Замена (Isolation Forest)** — обнаружено выбросов: **60**, заменены медианами

#### 3. Нестандартные признаки

- Извлечение `model_id` из URL (`purchase_link`)
- Regex: `model-(\d+)` → числовое поле

#### 4. Отбор признаков

| Метод | Алгоритм | Выбраны признаки |
|-------|----------|-----------------|
| **Filter** | SelectKBest (F-score) | `gdp_per_capita`, `avg_income`, `fuel_price` |
| **Wrapper** | RFE (LogisticRegression) | `ev_sales`, `charging_stations`, `avg_income` |
| **Embedded** | Lasso (L1) | `ev_sales`, `avg_income` |

### Выводы

1. **Масштабирование**: RobustScaler оптимален для данных с выбросами, так как использует медиану вместо среднего
2. **Выбросы**: Isolation Forest точнее IQR, но сложнее в интерпретации. Выбор метода зависит от контекста
3. **Нестандартные признаки**: URL и текстовые поля можно преобразовать в числовые через regex
4. **Отбор признаков**: 
   - Filter (SelectKBest) — быстрый, но не учитывает взаимосвязи
   - Wrapper (RFE) — точнее, но медленнее
   - Embedded (Lasso) — автоматический отбор с регуляризацией
